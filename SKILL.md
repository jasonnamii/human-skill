# Human Skill — 16축 행동 분석 엔진

16축 + 44메타원리로 인간행동의 심리메커니즘을 체계적으로 진단·예측·설계하는 도구. 각 축은 메커니즘→체인→자명명제로 분해되며, 메타원리는 다중 축 활성 시의 상호작용을 규정한다.

---

## ⛔ 절대 규칙

| # | 규칙 | 이유 |
|---|---|---|
| 1 | **스포크 없이 기억으로 실행 금지** — 해당 축/메타원리 상세 필요시 `@uses` 참조 강제 로드 | 16축×44원리를 기억만으로 정확 재현 불가 |
| 2 | **스크리닝 없이 상세 진입 금지** — §2 스크리닝 → 활성 축 식별 → 해당 스포크만 로드 | 전수 로드=토큰낭비+초점분산 |
| 3 | **메타원리 무시 금지** — 복수 축 활성 시 반드시 메타원리로 축간 상호작용 해석 | 개별 축 나열=단편적. 메타원리가 인과구조 결정 |

## §1. 모드 자동 판별

| 입력 유형 | 모드 | 판별 |
|---|---|---|
| 관찰된 행동·현상 | **진단** | "왜 이런 행동?" → 과거·현재 인과 추적 |
| 대상의 상태·맥락 | **예측** | "이 조건에서 어떤 행동?" → 미래 확률 추정 |
| 원하는 행동·반응 | **설계** | "어떤 메커니즘 활성화?" → 트리거 역추적 |

**복합 시**: 진단→예측→설계 순서 파이프라인. 애매 시 형에 확인.

## §3. 메타원리 라우팅 — 활성 축 조합 → 상호작용

스크리닝에서 2개 이상 축 활성 시 Python 라우터 즉시 조회 (스포크 전체 로드 불필요).

### Python 라우팅 (권장 — 토큰 절약 + 정확도)

```bash
# 기본 라우팅 + 상세 자명명제·설계원칙
python scripts/meta_router.py --full 축1 축5 축H

# 축 전문 조회
python scripts/meta_router.py --axis 축1
python scripts/meta_router.py --axis 축E --table

# 유틸리티
python scripts/meta_router.py --mode 진단      # 모드 프로토콜
python scripts/meta_router.py --loops          # 자기강화 루프만
python scripts/meta_router.py --hubs           # 허브 축 요약
python scripts/meta_router.py --all            # 전체 44개 매트릭스
```

**폴백**: Python 불가 시 `references/` 스포크 파일에서 수동 조회.

### 메타원리 요약 (44개 = 6계열)

자기강화 루프 6개: M5, CM7, SM6, BM3, BM4, EmoM5
설계 최우선 3: SM6(상승/하강) → EmoM5(조절-충족) → BM3(습관-정체성)

저활용 34개 활성화: `references/low-usage-meta-guide.md` 로드 (도메인·조합·극단 3분류 + 트리거)

## §5. 모드별 실행 요약

**진단**: 행동기술 → 스크리닝 → 체인추적 → 메타원리매핑 → 인과조합 → **리포트 변환**
**예측**: 상태파악 → 축식별 → 메타원리시뮬 → 허브파급 → 시나리오분기 → **확률근거부착** → **리포트 변환**
**설계**: 목표정의 → 역추적 → 트리거설계 → 시너지조합 → 리스크체크 → **리포트 변환**

### 체인 번호 특정 규칙 (전 모드 공통)

인과·역추적·시뮬에서 **핵심 경로의 축 1~2개는 체인 번호 특정** (--detail/--axis로 확인). 형식: `[축X 체인①] → [메타원리Z] → [축W 체인②]`

### 예측 모드 확률 근거 필수

확률 제시 시 ①유사 선행 사례 1건+ ②기저율 제시 ③확률 근거 태그(`[통계]` `[유추]` `[전문가판단]` `[모델추정]`) 3단계 필수.

## §6. 축 체계 계층 구조

```
[기저층 — 진화적 설계]
축G(위협관리) → 축1·3·5·6·C·H·I의 궁극적 원인
축F(번식전략) → 축4·E의 궁극적 원인

[인지 엔진]
축1(인지자원) ← 기저 제약 → M1 → 축5(감정선행)
축2·3·4·6(속성대체·손실회피·스토리·기억왜곡)
축7(판단잡음) ← 독립 오류원

[감정 과정 — 허브①]
축H(감정구성·조절) ← 7축과 교차하는 중심 허브

[동기 엔진]
축D(인과소재) ↔ 축E(욕구충족) ← SM6 쌍방향 강화

[사회적 메커니즘]
축A(호혜) ↔ 축B(자기상) ↔ 축C(정체성)

[행동 실행 — 허브②]
축I(행동루프·습관) ← 8축과 교차하는 실행 허브
```

**3대 허브**: 축H(감정, 8접합)·축I(행동, 8접합)·축E(동기, 6접합) — 상세: `python scripts/meta_router.py --hubs`

**진화축(F·G) 주석 규칙**: 활성 시 진단/예측에서 1줄 자동 주석("이 행동의 진화적 기원: [⠀] → 현대 맥락 불일치: [⠀]"). 설계에서는 "설계 레버 아님" 명시 + 근접축(F→4·E / G→1·3·5·6·C·H·I)에서 설계.

## Gotchas

| # | 안티패턴 | 올바른 접근 |
|---|---|---|
| 1 | 16축 전체 상세 분석 | 스크리닝 후 활성 축 3~5개 집중 |
| 2 | 개별 축 나열·메타원리 무시 | 복수 축 활성 시 반드시 메타원리로 상호작용 해석 |
| 3 | 축을 독립적으로 취급 | 축은 계층 구조(§6). 기저→인지→감정→동기→사회→행동 흐름 |
| 4 | 진화축(F·G) 직접 설계 대상화 | 진화축=궁극 설명 도구. 설계 레버는 근접 축 |
| 5 | 허브 축(H·I·E) 무시 | 허브가 다른 축을 연결. 미고려=인과 구조 불완전 |
| 6 | 메타원리 방향 무시(→ vs ↔) | 단방향과 쌍방향은 설계 함의 다름. ↔=자기강화 루프 가능 |
| 7 | 자명명제를 직접 인용 | 축→체인→메타원리 수준 분석만 |
| 8 | 스크리닝·메타원리 테이블을 최종 산출물로 | §4.5 리포트 변환 필수. 데이터 시트→서사 리포트 |
| 9 | "히트시키려면?" 요청에 human-skill 적용 | 히트·자극설계·전파구조 → hit-skill. human-skill은 행동 인과 분석·예측·설계 전용 |
