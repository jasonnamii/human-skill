# 휴먼스킬 3가지 운용 모드 가이드

허브 SKILL.md §5에서 제시된 3가지 모드(진단·예측·설계)의 상세 실행 프로토콜. 각 모드의 단계별 절차, 출력 형식, 실행 체크리스트를 제공합니다.

---

## 1. 진단 모드 — "왜 이런 행동이 발생하는가"

**적용 시점**: 관찰된 행동·현상의 인간 메커니즘 수준 원인 추적

### 실행 프로토콜

| 단계 | 작업 | 체크포인트 |
|---|---|---|
| ① 관찰 기술 | 행동/현상을 구체적으로 기술 (누가, 어떤 상황에서, 무엇을) | 추상적 표현 없음 |
| ② 16축 스크리닝 | SKILL.md §2 핵심 질문으로 각 축 활성/비활성 판정 | 활성 축 3~5개 선별 |
| ③ 활성 축 심화 | 활성화된 축의 스포크 로드 → 내부 인과 체인 중 트리거 식별 | 체인 시작점 명확 |
| ④ 메타원리 매핑 | 활성 축 쌍의 메타원리 확인 (meta-principles.md) → 축간 상호작용 해석 | 방향성(→ vs ↔) 확인 |
| ⑤ 인과 체인 조합 | [축X 체인Y] → [메타원리Z] → [축W 체인V] 형태로 최종 설명 구성 | 2~3개 연쇄 가능 |
| ⑥ 확인 질문 | "이 인과가 관찰된 행동을 충분히 설명하는가?" → 불충분 시 재스캔 | 누락 축 검토 |

### 출력 형식

| 구성 요소 | 내용 |
|---|---|
| **스크리닝 매트릭스** | 16축 활성/비활성 표 + 활성화 근거 1줄 |
| **핵심 인과 체인** | [축X 체인Y] → [메타원리] → [축W 체인V] 형태 |
| **메커니즘 설명** | 2~3문장 인과 설명 + 확신도 표기 |
| **추가 관찰 필요** | 인과 확인을 위해 더 관찰할 것 |

### 사용 예시

**관찰**: 팀원이 프로젝트 반대 의견을 내지 않고 침묵한다

| 결과 | 설명 |
|---|---|
| **활성 축** | 축1(인지 부하), 축A(의무감), 축B(자기상 일관성), 축D(자율성 훼손) |
| **인과 체인** | 축1(S1 기본값) → CM1(무비판 실행) → 축A 의무감 순응<br>축B(자기상 고착) → CM2(이탈 불가)<br>축D(통제 환경) → SM4(반항 대신 순종 선택) |
| **메커니즘** | 인지 부하 상태에서 형성된 의무감이 무비판적 순응을 유발하고, 이전 동의 이력이 자기상을 고착시켜 반대 의견 표출의 심리적 비용을 높였다. 동시에 통제적 환경이 자율성을 훼손하여 순종이 선택되었다. |
| **확신도** | 중상 (맥락 정보 추가 시 상향 가능) |

## 2. 예측 모드 — "어떤 조건에서 어떤 행동이 나타나는가"

**적용 시점**: 미래 행동 예측 또는 "이 조건 변화 시 행동 변화" 시뮬레이션

### 실행 프로토콜

| 단계 | 작업 | 체크포인트 |
|---|---|---|
| ① 현재 상태 4차원 파악 | 인지(자원/부하), 감정(세분화도), 사회(관계/의무감), 동기(내/외적 구분) | 각 1줄 이상 |
| ② 축 활성화 예측 | 현재 상태에서 활성화될 가능성 높은 축 선별 | 확률(높/중/낮) 표기 |
| ③ 메타원리 시뮬레이션 | 활성 축 쌍의 메타원리로 상호작용 예측 + 자기강화 루프 확인 | M5, CM7, SM6, BM3, BM4, EmoM5 검토 |
| ④ 허브 축 점검 | 축H(감정)·축I(행동)·축E(동기) 중 활성화된 허브의 파급 효과 | 다른 축 연쇄 여부 |
| ⑤ 행동 확률·강도 추정 | "이 조건에서 이 행동이 나타날 확률은 [높/중/낮]" | Top 2~3 행동 순위 |
| ⑥ 시나리오 분기 | "변수 X 변화 → 축 Y 전환 → 행동 Z" (2~3개 시나리오) | 실행 가능한 변수만 |

### 출력 형식

| 구성 요소 | 내용 |
|---|---|
| **현재 상태** | 인지·감정·사회·동기 각 1줄 |
| **활성 축** | 축별 확률 표 (높/중/낮) |
| **자기강화 루프** | 루프 존재 여부 + 가속/감속 방향 |
| **행동 예측** | Top 2~3 행동 + 확률 |
| **분기 시나리오** | 변수 변화 시 축 전환 경로 |

### 사용 예시

**예측**: 신규 구독 서비스 가입 후 3개월 시점 이탈 확률

| 결과 | 설명 |
|---|---|
| **현재 상태** | 인지(자동 결제로 저비용), 감정(초기 흥분 감쇠), 사회(커뮤니티 미형성), 동기(외적→내면화 불완전) |
| **활성 축** | 축3(현상유지) 높음, 축I(습관) 중, 축E(욕구) 높음, 축B(자기상) 낮음 |
| **루프** | BM4(보상→욕구충족 경로 불일치) 발동 → 이탈 가속 |
| **행동 예측** | ①이탈(확률 높음) ②대기(확률 중) ③재구독(확률 낮음) |
| **시나리오** | 변수①: 커뮤니티 활성 → 축C 활성 → 이탈 비용↑<br>변수②: 주간 습관 루프 → 축I 강화 → 유지 가능성↑ |

## 3. 설계 모드 — "어떤 메커니즘을 활성화해야 하는가"

**적용 시점**: 특정 행동·반응 유도를 위한 트리거·조건·환경 설계

### 실행 프로토콜

| 단계 | 작업 | 체크포인트 |
|---|---|---|
| ① 목표 행동 정의 | "[상황]에서 [대상]이 [행동]을 하게 하고 싶다" | 관찰 가능하고 측정 가능 |
| ② 역추적 | 목표 행동을 유발하는 축·체인을 역방향으로 추적 | 행동 → 축 → 체인 → 트리거 |
| ③ 트리거 설계 | 체인 시작점을 활성화할 자극·조건·환경 설계 | 구체적이고 실행 가능 |
| ④ 시너지 축 조합 | 메타원리로 2개 이상 축의 같은 방향 강화 가능성 | CM7, BM4, BM5 등 검토 |
| ⑤ 리스크 체크 | 의도하지 않은 축 활성화 + 부작용 루프 + 윤리 경계 | 자율성·정체성·혐오 검토 |
| ⑥ 실행 설계서 | 시간순 액션 플랜: 트리거 → 축 → 메타원리 → 결과 | 타이밍·순서·강도 명시 |

### 출력 형식

| 구성 요소 | 내용 |
|---|---|
| **목표 행동** | 1문장 정의 |
| **역추적 경로** | 축→체인→트리거 조건 |
| **트리거 설계** | 구체적 자극/조건/환경 |
| **시너지 조합** | 메타원리 기반 축 강화 |
| **리스크 + 차단책** | 부작용 축 식별 + 대응책 |
| **액션 플랜** | 시간순 실행 단계 |

### 사용 예시

**목표**: 신규 사용자가 온보딩 후 1주일 내 핵심 기능을 3회 이상 사용

| 결과 | 설명 |
|---|---|
| **역추적** | 반복 사용(축I 루프) ← 즉각 성공 경험(축E) + 마찰 제거 |
| **트리거 설계** | ①최초: 즉각 성공(축E 유능감)<br>②2일차: 알림(외부 갈망)<br>③3일차: 소셜 피드백(축A 호혜 + 축C 소속감) |
| **시너지** | BM4(보상→욕구충족) + BM5(자율선택→의지력) + CM4(감정→사회강화) |
| **리스크** | SM2(알림이 통제적 → 자율성 훼손) → 차단: 정보적 프레이밍("새 기능" vs "아직 안 해봤다") |
| **액션** | Day 1: 성공 경험 즉시 제공<br>Day 2: 정보 알림<br>Day 3-7: 점진적 사회적 강화 |

## 4. 모드 간 전환 규칙

| 전환 | 트리거 | 사용 방법 |
|---|---|---|
| **진단 → 설계** | "원인을 알았으니 바꾸고 싶다" | 진단의 약점 축을 설계의 역추적 입력으로 사용 |
| **진단 → 예측** | "이 원인이 지속되면 어떻게 될까" | 진단의 활성 축+메타원리를 예측 시뮬레이션 입력으로 사용 |
| **예측 → 설계** | "이 예측을 바꾸고 싶다" | 예측의 분기 시나리오 중 원하는 방향의 변수를 설계 입력으로 사용 |
| **설계 → 진단** | "설계를 적용했는데 의도대로 안 됐다" | 실제 결과를 진단으로 분석 → 누락 축/메타원리 발견 |

## 5. 공통 주의사항

| # | 주의 | 이유 | 실행 기준 |
|---|---|---|---|
| **1** | 3~5개 핵심 축에 집중 | 16축 전수 분석은 초점 분산 | 스크리닝으로 선별 후 심화만 수행 |
| **2** | 메타원리 방향 확인 | 단방향(→) vs 쌍방향(↔) 여부에 따라 루프 가능성 달라짐 | 모든 메타원리마다 방향성 명시 |
| **3** | 허브 축 필수 점검 | 축H·I·E의 파급 효과가 다른 축 활성화 연쇄 유발 | 활성 축 중 허브 포함 시 "연쇄 효과" 섹션 추가 |
| **4** | 진화축은 설계 대상 아님 | 축F·G는 이해·예측에만 사용, 조작 불가 | 설계 시 축F·G를 직접 트리거로 설정 금지 |
| **5** | 윤리적 경계 | 자율성 훼손·정체성 조작·혐오 이용 가능하지만 권장되지 않음 | 설계 리스크 체크 필수 |
| **6** | 확신도 표기 | 메타원리 기반 추론의 불확실성 명시 | 모든 결론에 "높/중/낮" 또는 "확실성 %" 추가 |

## 6. 빠른 참조 — 모드별 핵심 질문

### 진단 모드
- 이 행동은 어떤 축들이 조합되었을 때 일어나는가?
- 그 축들을 트리거한 자극은 무엇인가?
- 활성 축 사이에 메타원리로 강화되는 부분이 있는가?

### 예측 모드
- 현재 상태에서 어떤 축이 활성화될 가능성이 높은가?
- 활성화된 축들 사이에 자기강화 루프가 있는가?
- 이 루프가 가속되려면 어떤 조건이 지속되어야 하는가?

### 설계 모드
- 목표 행동을 하려면 어떤 축이 활성화되어야 하는가?
- 그 축을 활성화하려면 어떤 자극이 필요한가?
- 그 자극이 원치 않는 축도 활성화할 위험이 있는가?
