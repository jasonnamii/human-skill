# 휴먼스킬 리포트 템플릿

§4.5 리포트 변환 시 이 구조로 재구성한다.

---

## 본문 — 3블록 구조

### 블록1: 핵심 판정 (원페이저)

활성 축·메타원리·허브 연결의 핵심을 한 문단으로 압축한다.
"왜 이런 행동이 나타나는가" / "어떤 행동이 예측되는가" / "어떤 메커니즘을 건드려야 하는가"의 답을 3~5문장으로 단언.

```
[작성 가이드]
- 진단: "이 행동의 핵심 동인은 [축X]와 [축Y]의 상호작용이다. [메타원리Z]에 의해..."
- 예측: "현재 조건에서 [행동A]가 발생할 확률이 높다. 이유는..."
- 설계: "[목표 행동]을 유도하려면 [축X]의 [체인①]을 트리거해야 한다."
- 최소 3문장. 확률 수치 제시 시 근거 태그 필수
```

### 블록2: 메커니즘 서사 (인과 해설)

활성 축 2~4개의 상호작용을 인과 문단으로 풀어쓴다.
축명·메타원리명은 괄호 레이블로만 사용. "축1이 축5를 활성화하여…" 식의 코드형 나열 금지.

```
[작성 가이드]
- 각 활성 축 = 1문단. 문단 구조: 입력물 근거 인용 → 메커니즘 → 효과/리스크
- 메타원리는 축간 연결 문장에서 자연스럽게 언급 ("감정이 먼저 판단을 내리고(M1), 그 판단이...")
- 허브 축(H·I·E) 연결은 별도 문단으로 ("이 모든 경로가 결국 [행동 루프]에서 수렴한다...")
- 진화축(F·G) 활성 시: 주석 1문장 삽입 ("진화적으로는 [X]에서 비롯되나, 현대 맥락에서는...")
- 최소 3문장/블록. 표·리스트 단독 사용 금지
```

### 블록3: 시사점 / 설계안

모드별로 분기한다.

```
[작성 가이드]
- 진단: 블록3 생략 (블록1+블록2로 종결)
- 예측: 시나리오 분기 서술 → 확률+근거 → "만약 [조건]이면 [행동B]로 전환될 수 있다"
- 설계: 트리거 설계 → 시너지 조합 → 리스크 체크. 조건부 형식("만약 ~라면") 필수
- 최소 3문장. 설계 모드에서 액션 플랜은 서사 후 별도 블록
```

## 부록 (본문 하단 배치)

본문의 근거 데이터. 필요시 참조용.

### A. 스크리닝 결과
16축 활성/비활성 1줄 판정 원본 (§2).

### B. 메타원리 매핑
활성 메타원리 목록 + 축간 방향성(→/↔) + 자기강화 루프 해당 여부.

### C. 체인 상세
핵심 경로의 체인 번호·자명명제 (§4에서 추출).

### D. 허브 축 연결맵
축H·I·E 접합 관계 요약.

### E. 확률 근거 (예측 모드만)
기저율 + 유사 사례 + 근거 태그.
